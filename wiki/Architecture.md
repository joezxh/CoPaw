## 核心组件

### 1. Agent 框架

CoPaw 基于 **AgentScope** 的 ReActAgent 构建：

```python
class CoPawAgent(ToolGuardMixin, ReActAgent):
    """CoPaw 智能体
    
    继承关系：
    - ReActAgent: AgentScope 的 ReAct 框架实现
    - ToolGuardMixin: 工具守卫安全拦截
    """
```

**ReAct 循环**:
```
User Query → Reasoning → Action → Observation → Reasoning → ... → Response
```

### 2. FastAPI 应用

提供 REST API 和 WebSocket 接口：

| 路由 | 功能 |
|------|------|
| `/api/agents` | 智能体管理 |
| `/api/chat` | 聊天接口 |
| `/api/skills` | 技能管理 |
| `/api/providers` | 模型提供商 |
| `/ws` | WebSocket 连接 |

### 3. 渠道系统

支持多种通信渠道的统一抽象：

```python
class BaseChannel(ABC):
    async def start(self)
    async def stop(self)
    async def send_message(self, message)
    async def receive_message(self) -> Message
```

**实现渠道**:
- Console (HTTP/WebSocket)
- DingTalk (Stream)
- Feishu (WebSocket)
- Discord (Bot API)
- Telegram (Bot API)
- QQ (OneBot)
- WeChat (HTTP API)
- Enterprise WeChat (WebSocket)
- iMessage (AppleScript)
- Matrix (Client API)

### 4. 技能系统

双层技能架构：

```
skill_pool/              # 共享池
├── skill.json          # 池清单
└── pdf/
    └── SKILL.md

workspaces/default/skills/  # 工作区副本
├── skill.json
└── pdf/
    └── SKILL.md
```

**技能加载流程**:
1. 解析 SKILL.md
2. 提取工具定义
3. 注册到 Toolkit
4. 处理依赖和环境变量

## 数据流

### 1. 消息处理流程

```
用户消息
    ↓
Channel 接收
    ↓
路由到 Agent
    ↓
ReAct 循环
    ↓
工具调用 (可选)
    ↓
记忆更新
    ↓
生成响应
    ↓
返回给用户
```

### 2. 定时任务流程

```
定时触发 (APScheduler)
    ↓
加载任务配置
    ↓
选择智能体
    ↓
执行任务
    ↓
发送结果到指定渠道
```

### 3. 技能广播流程

```
选择技能池技能
    ↓
选择目标工作区
    ↓
复制技能文件
    ↓
更新工作区清单
    ↓
重新加载技能
```

## 性能优化

### 1. 模型调用优化

- **速率限制**: 滑动窗口 QPM 限制
- **重试机制**: 指数退避重试
- **流式响应**: 减少首字延迟
- **模型路由**: 根据任务选择合适模型

### 2. 记忆优化

- **自动压缩**: 超过阈值自动压缩
- **语义检索**: 基于 ReMe 的语义搜索
- **分层存储**: 短期记忆 + 长期记忆

### 3. 并发处理

- **异步 I/O**: 全异步架构
- **连接池**: 复用 HTTP 连接
- **任务队列**: 优先级队列管理
