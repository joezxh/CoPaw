## 简介
本指南面向首次接触 CoPaw 的用户，帮助你在最短时间内完成安装、初始化配置与首次使用。无论你是个人用户还是企业用户，都能在 3 分钟内启动控制台并开始对话。我们提供多平台安装方式与企业级部署选项，涵盖 pip、Docker、桌面应用与脚本安装，并说明首次登录的两种模式（普通用户与企业用户）差异。

## 系统要求与环境准备
- Python 版本：3.10 ≤ Python < 3.14（安装器会自动管理 Python 环境）
- 操作系统：Windows、macOS、Linux
- 网络：首次启动需要访问 PyPI 与可选的模型提供商
- 存储：首次运行会创建工作目录（默认位于用户主目录下的 .copaw），用于保存配置、会话与媒体文件
- 可选：浏览器访问控制台（默认端口 8088）

## 安装方式总览
CoPaw 提供多种安装方式，适合不同使用场景与技术水平：

- pip 安装（个人使用）：最直接，适合熟悉 Python 的用户
- 企业安装（多租户）：安装企业版依赖，支持 PostgreSQL、Redis
- Docker 部署：快速体验与容器化部署
- 桌面应用（Beta）：无需命令行，适合非技术用户
- 脚本安装：自动化安装，支持 extras 与版本选择

## 常见问题与故障排查

- 端口被占用
  - 解决：更换端口启动：`copaw app --port 8089`
- 无法连接数据库（企业版）
  - 检查 PostgreSQL/Redis 容器是否运行，使用 `docker ps` 查看
  - 使用 `psql` 或 `redis-cli` 连接测试
- 控制台无法访问
  - 确认前端资源已构建或通过包内资源提供
  - 检查静态文件目录解析逻辑
- Windows 执行策略限制
  - PowerShell 需调整执行策略或使用脚本安装方式
