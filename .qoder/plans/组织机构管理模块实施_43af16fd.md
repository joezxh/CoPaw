# 组织机构管理模块实施计划

## 背景

当前系统中存在两个组织管理相关的模块：
1. **UserGroup（用户组）**：扁平结构，无层级关系
2. **Department（部门）**：树形结构，已有 parent_id、level 字段和递归 CTE 查询

用户需求：移除 UserGroup，将 Department 改造为完整的组织机构管理模块。

## 实施范围

### 移除部分（UserGroup 模块）
- 数据库模型：`UserGroup`、`UserGroupMember`
- 数据库表：`sys_user_groups`、`sys_user_group_members`
- 后端 API：`user_groups.py` 路由
- 前端页面：`Groups/GroupList.tsx`
- 前端 API：`enterprise-groups.ts`
- 任务模型字段：`Task.assignee_group_id`
- 菜单项：用户组相关配置
- 初始化脚本中的用户组表创建

### 增强部分（Department → Organization）
- 扩展现有 Department 模型的 API 功能
- 创建前端组织机构管理页面
- 实现树形展示和递归遍历
- 支持在组织节点中添加/管理用户成员
- 更新菜单和路由配置

### Step 7: 增强 Department API 功能

**文件：`src/copaw/app/routers/departments.py`**

增强内容：
1. **改进树形查询**：返回嵌套的树形结构而非扁平列表
   ```python
   def build_tree(depts):
       """将扁平列表转换为嵌套树形结构"""
       dept_map = {d['id']: {**d, 'children': []} for d in depts}
       tree = []
       for dept in depts:
           if dept['parent_id']:
               dept_map[dept['parent_id']]['children'].append(dept_map[dept['id']])
           else:
               tree.append(dept_map[dept['id']])
       return tree
   ```

2. **新增成员管理 API**：
   - `POST /{dept_id}/members` - 批量添加成员
   - `DELETE /{dept_id}/members/{user_id}` - 移除成员
   - `GET /{dept_id}/members/count` - 获取成员数量（含子部门）

3. **新增统计信息**：
   - `GET /{dept_id}/stats` - 获取部门统计信息（成员数、子部门数）

4. **改进删除逻辑**：
   - 支持级联删除或移动子部门选项

## 风险和注意事项

1. **数据丢失风险**：删除用户组表会丢失所有用户组数据，已确认不需要迁移
2. **外键依赖**：必须先删除 `ai_tasks.assignee_group_id` 外键，才能删除 `sys_user_groups` 表
3. **菜单权限**：组织机构的权限码从 `group:read` 改为 `org:read`，需要更新权限配置
4. **向后兼容**：如果有外部系统依赖用户组 API，需要通知相关方
5. **前端组件复用**：可以考虑复用 Departments 的现有 API，只需增强功能
